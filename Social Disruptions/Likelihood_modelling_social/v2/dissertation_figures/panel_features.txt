MODELLING PANEL — FEATURE REFERENCE
================================================================================
Panel coverage : 39 countries  x  daily  x  2018–2020
Panel file     : v2/data/interim/modelling_panel_gdelt.parquet
Feature source : v2/src/train_backtest.py  (FEATURES_M0 … FEATURES_M4_ADD)

Targets (binary, forward-looking)
----------------------------------
  protest_7d      1 if any protest event detected in next 7 days
  strike_7d       1 if any labour strike detected in next 7 days
  strike_30d      1 if any labour strike detected in next 30 days

  NOTE: protest_30d excluded — near-universally 1 for reliably-monitored
        countries (≥94% positive rate), leaving no discriminatory signal.

  Labels are generated via Positive-Unlabelled (PU) learning:
    y = 1   Confirmed positive  (event detected in any country)
    y = 0   Reliable negative   (reliably-monitored country, no event)
    y = NaN Unlabelled          (poorly-monitored country, zero not trusted)

  Reliability threshold: ≥4% of country-days with medium/high GDELT coverage.

================================================================================
DROPPED / EXCLUDED FEATURES  (and why)
================================================================================

  protest_30d target         : 93–94% positive rate for reliably-monitored
                               countries — no discriminatory signal.

  earnings_monthly / _z      : 92% missing across the panel — too sparse.

  *_z variants of annual WDI : Expanding z-score with min_obs=3 means NaN
                               throughout 2018–2019 (entire training period);
                               only valid in 2020 (test year) — not useful.

  economic_stress_index      : Google Trends-based; 51% missing, cannot be
  labour_conflict_index        completed for all 39 countries.
  protest_mobilisation_index

  All features shifted by forecast horizon (7 or 30 days) before training
  to prevent information leakage from the target window into features.
